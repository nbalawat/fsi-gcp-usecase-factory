Option A · density

↗ open standalone
density ★★☆☆☆ motion · minimal aff · stage-rail-drill metaphor · process-as-rail persona · Executive (CFO/CRO) finalizing a quarterly CECL allowance run with a 30-day OCC clock

Option A treats the run as a SINGLE NUMBER you sign, supported by a four-stage horizontal rail at the top. The rail IS the page: every pixel below the rail responds to which stage is open. Sparse density for the hero (one allowance, one delta, one exceptions count, one OCC clock) so the executive reads the run's state at a glance. Click a stage and the dense numeric ledger (12 segments × 4 forecast quarters × bps) unfolds inline — that's where the analyst-level detail lives, behind the executive surface.

reuse 16 + CFO/CRO at-a-glance comprehension — three numbers and a clock+ Stage rail makes "where are we in the OCC 30-day window" answerable in one second+ Dense per-segment ledger available without leaving the page+ Approval flow surfaces the irrevocable GL posting + qual-overlay context inline − Story / transcript view of HOW the allowance was produced is collapsed (option D's territory)− Per-cell drill-into-event (which agent set this PD) is not first-class — the audit trail lives on the run-detail page only− No live SSE stream — the rail status is derived from events at render time (Rule 13 trade-off: this view is intended for executive review, not in-flight monitoring)− Segment-level human override UI is intentionally read-only here; overrides happen earlier in the workflow

Option C · affordance

↗ open standalone
density ★★★☆☆ motion · minimal aff · inline-per-segment-row metaphor · row-as-disposition-unit persona · Risk Analytics Lead approving Q2 segment methodologies

Option C treats each CECL segment as a one-row dispatch unit. The row itself carries the PD/LGD/EAD inputs that produced its reserve, the reserve number, the methodology owner, AND the action that segment enables — "Approve methodology" (reversible) or "Open variance Q&A" inline. No modal, no sticky bottom bar, no separate review screen. The user's eye never leaves the row to dispose of the row. CFO attestation is the SINGLE action that escapes inline. It is irrevocable per the canvas hitl_gates contract, regulator-visible, and lives on /approval where the run rolls up into one figure the CFO

reuse 12 + Disposition speed — one click on the row, no context switch+ Audit explainability — every action records on the same row that surfaces the inputs that triggered it+ Reversibility — segment-methodology approvals are reversible right up until CFO attest+ Reuse — seven shared primitives carry the foundation (AppShell, BreadcrumbNav, MetricStrip, RegulatoryClock, StatCard, StatusBadge, ApprovalGate); novelty is in the row composition − Wide rows — the 5-column row layout (ring + identity + inputs + reserve + action) degrades on narrow viewports; below 1024px collapses to a stack which is less efficient for skimming− No spatial pipeline-stage rail — process is implicit in the segment-row verdict badges, not a visual left-to-right axis− Variance Q&A is a single textarea, not a structured form — banker freedom over field-level validation− The audit ledger lives BELOW the rows (full-width section), not in a right rail — banker sees rows first, audit-as-SOP after
